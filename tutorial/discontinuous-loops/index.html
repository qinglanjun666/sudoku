Discontinuous Loops

1. What You Will Master

In this section, you will learn the Discontinuous Loop (often called a "Nice Loop" with a discontinuity). You will discover what happens when a chain wraps around the board and connects back to the starting cell, but fails to maintain the perfect alternating rhythm. You will learn to spot the two outcomes: the Self-Contradiction (which kills the candidate) and the Self-Validation (which sets the candidate).

2. Main Content

The Concept

A Discontinuous Loop is an AIC that returns to the Start Node but ends on the "wrong" note.

Continuous Loop: Start = A ... B - Start (Ends with a Weak link, perfectly alternating).

Discontinuous Loop: Start = A ... B = Start (Ends with a Strong link, creating a clash).

Because the rhythm is broken at the meeting point, the Start Node gets hit with two identical logical states (Two Strongs or Two Weaks).

Type 1: The Loop of Death (Elimination)

Structure: Start - A = B ... C - Start

The Clash: The loop starts with a Weak Link out of the candidate and ends with a Weak Link coming back into it.

The Logic: Assume Start is TRUE. Follow the chain: A is False → B is True... Eventually, the chain proves that C is True. But C has a Weak Link to Start (they share a house). If C is True, Start MUST BE FALSE.

The Paradox: "If Start is True, then Start is False."

Conclusion: Since being True leads to a contradiction, Start MUST be False. You can eliminate the candidate immediately.

Type 2: The Loop of Life (Forcing the Digit)

Structure: Start = A - B ... C = Start

The Clash: The loop starts with a Strong Link and ends with a Strong Link.

The Logic: Assume Start is FALSE. Follow the chain: A must be True → B is False... Eventually, the chain proves that C is True. But C has a Strong Link to Start (e.g., they are the only two in a row). If C is True... wait, the logic here is: "If Start is False, then C forces Start to be True."

The Paradox: "If Start is False, then Start is True."

Conclusion: Being False is impossible. Therefore, Start MUST be True. You can fill in this number with a pen.

Visual Check: "Head to Tail"

When you build a chain that lands back on the start:

  • Two Weaks touching Start? → Start is False (Delete it).
  • Two Strongs touching Start? → Start is True (Solve it).
  • One Strong, One Weak? → It's a Continuous Loop (See Section 14.2).

3. Section Summary

Discontinuous Loops are the ultimate judge. They allow a candidate to testify against itself. If a number's own existence implies its non-existence (Type 1), it commits logical suicide. This technique effectively ends "Chaining" strategies.
