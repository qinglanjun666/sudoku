Sue de Coq (SDC)

1. What You Will Master

In this section, you will learn the Sue de Coq (SDC) technique. You will learn to spot a specific pattern where a Row and a Box intersect. You will understand how to use the "Intersection Cells" as a pivot to cause a simultaneous "Flash Freeze" in both the Row and the Box, eliminating candidates from both areas at once.

2. Main Content

The Concept: The Venn Diagram

Imagine two circles overlapping:

  • Circle A: A Row (Linear Logic).
  • Circle B: A Box (Box Logic).
  • The Overlap: The Intersection (2 or 3 cells shared by both).

The SDC technique occurs when the candidates inside the Intersection are too crowded to satisfy both the Row and the Box separately, so they force the external candidates to lock into place.

The Components

To find an SDC, look for three sets of cells involving a specific list of candidates (e.g., {1, 2, 3, 4, 5}):

  • The Intersection (Pivot): Two cells in the overlap (e.g., Row 1, Box 1) containing {1, 2, 3, 4}.
  • The Line Wing: A cell in the same Row outside the Box containing {1, 5}.
  • The Box Wing: A cell in the same Box outside the Row containing {3, 4}.

The Math (Capacity Check)

Total Cells: 4 cells (2 Pivot + 1 Line Wing + 1 Box Wing).

Total Candidates: 5 ({1, 2, 3, 4, 5}).

SDC Condition: The candidates in the Pivot must equal the number of Pivot cells plus the number of sets it connects (usually 2). The Pivot cells are "almost locked" in two directions at once.

The Logic: "The Split"

Look at the Pivot {1, 2, 3, 4}. It cannot satisfy everything. It acts as a gatekeeper.

  • If the Pivot consumes the Row logic (e.g., uses 1s and 2s), it starves the Box.
  • If the Pivot consumes the Box logic (e.g., uses 3s and 4s), it starves the Row.

Result: The Wings must step in. The Line Wing takes the pure line candidates (e.g., 5). The Box Wing takes the pure box candidates (e.g., 3 and 4).

The Elimination

Because the candidates are tightly constrained between these three groups:

  • In the Row: Any candidate appearing in the Line Wing (e.g., 1 or 5) can be eliminated from the rest of the Row (excluding the Pivot).
  • In the Box: Any candidate appearing in the Box Wing (e.g., 3 or 4) can be eliminated from the rest of the Box (excluding the Pivot).

You effectively treat Intersection + Line Wing as a locked set, and Intersection + Box Wing as a locked set simultaneously.

Simplified Recognition

Don't do the math. Look for this shape:

  • Focus on an intersection (Row entering a Box). Are there 2 cells there with many candidates (e.g., {A, B, C, D})?
  • Look left/right (Row): Is there a cell with a subset of those (plus maybe one new one, E)?
  • Look up/down (Box): Is there a cell in the box with the other subset?
  • Fire: If yes, you likely have SDC. Eliminate candidates from the rest of the row and box.

3. Section Summary

Sue de Coq is a "two birds with one stone" technique. It cleans a Row and a Box in one logical step. While formally seen as an intersection of two ALS sets, it’s best viewed as a capacity crisis in the intersection: the pivot cells are under too much pressure, forcing the wings to become locked sets instantly.
